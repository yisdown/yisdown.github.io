# 0xVoid — Cybersecurity Blog

A Jekyll-powered cybersecurity blog with a purple/black terminal aesthetic. Designed for CTF write-ups, vulnerability research, and security articles.

---

## 🚀 Quick Start

### Prerequisites
- Ruby 3.x
- Bundler (`gem install bundler`)
- Git

### Local development

```bash
# Clone your repo
git clone https://github.com/YOURUSERNAME/YOURUSERNAME.github.io
cd YOURUSERNAME.github.io

# Install dependencies
bundle install

# Serve locally (with live reload)
bundle exec jekyll serve --livereload

# Open http://localhost:4000
```

## ✍️ Writing a Post

Create a file in `_posts/` with the format `YYYY-MM-DD-title.md`:

```markdown
---
layout: post
title: "HackTheBox — MachineName Write-up"
date: 2024-06-15
category: CTF          # CTF | Web | Pwn | Reversing | Crypto | OSCP | Malware | Network
difficulty: Easy       # Easy | Medium | Hard | Insane
platform: HackTheBox   # HackTheBox | TryHackMe | PicoCTF | etc.
featured: false        # Set true to feature on homepage
tags: [htb, linux, privesc]
excerpt: "One sentence summary for cards and SEO."
---

## Section Title

Your content here...
```

## 🎨 Customization

### Colors & Theme

All colors are in `_sass/_variables.scss`. The key variables:

```scss
$purple-bright: #b06aff;   // Primary accent
$purple-mid:    #8844dd;   // Hover/borders
$bg-void:       #0a0a0f;   // Page background
$green-neon:    #39ff8a;   // CTF/success
$red-alert:     #ff3366;   // Pwn/danger
$cyan-accent:   #00d4ff;   // Web/info
```

### Adding a new category

1. Add the tag CSS in `_sass/_variables.scss` under `.tag`:
   ```scss
   &--yourcat { color: #yourcolor; border-color: rgba(#yourcolor,0.4); background: rgba(#yourcolor,0.08); }
   ```
2. Add the icon in `_config.yml` under `categories_icons`:
   ```yaml
   yourcat: "🔥"
   ```
3. Use it in post front matter: `category: Yourcat`

### Adding pages

Create any `.html` or `.md` file in the root with:
```yaml
---
layout: default
title: "Page Title"
permalink: /your-path/
---
```

Then add it to the nav in `_config.yml`:
```yaml
nav:
  - title: "Your Page"
    url: "/your-path/"
```

## 🌐 Deploying to GitHub Pages

1. Create a repo named `YOURUSERNAME.github.io`
2. Push this project to `main` branch
3. Go to **Settings → Pages → Source** and select **GitHub Actions**
4. The `.github/workflows/deploy.yml` handles everything automatically

Every push to `main` triggers a rebuild and deploy.

### Custom domain (optional)

Add a `CNAME` file to the repo root:
```
yourdomain.com
```
Then configure your DNS with a CNAME record pointing to `YOURUSERNAME.github.io`.

## 🛠️ Troubleshooting

| Problem | Fix |
|---------|-----|
| CSS not loading | Check `url:` in `_config.yml` matches your GitHub Pages URL |
| Fonts not loading | Ensure internet access; fonts load from Google Fonts |
| Build fails on GitHub | Check Ruby version in `.github/workflows/deploy.yml` |
| `bundle install` fails | Run `gem update --system` then retry |
